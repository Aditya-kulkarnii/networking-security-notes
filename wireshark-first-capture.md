# Wireshark — First Packet Capture Session

## Setup
- OS: Kali Linux (running in VirtualBox VM)
- Connected to: WiFi network
- Tool: Wireshark (pre-installed on Kali Linux)

---

## What I Did

### Step 1 — Started a live capture
Opened Wireshark, selected the WiFi network interface,
and clicked the blue shark fin button to begin capturing.

Immediately saw hundreds of packets being captured in real time
from multiple sources and destinations across the internet.

### Step 2 — Observed live traffic
Opened a browser inside the VM and visited a website.
Wireshark instantly showed the packets being generated by
that specific browsing activity — I could see the connection
being formed in real time.

Every packet showed:
- Source IP address (where it came from)
- Destination IP address (where it was going)
- Protocol being used (TCP, UDP, DNS, HTTP etc.)
- Length of the packet
- Info about what the packet was doing

### Step 3 — Applied filters to isolate specific traffic

**TCP filter** — typed `tcp` in the filter bar.
Result: Only TCP packets were displayed. All UDP, DNS,
and other protocol packets disappeared. Made it much
easier to focus on one type of traffic at a time.

**UDP filter** — typed `udp` in the filter bar.
Result: Only UDP packets were displayed. Saw how much
background UDP traffic exists even when not actively doing anything.

---

## Key Observations

1. The network is never silent — even when I was not actively
   browsing, packets were constantly being sent and received
   in the background by the OS and applications.

2. Filters make Wireshark usable — without filters, the raw
   capture is overwhelming. Learning filters is essential.

3. You can see protocols in action — TCP connections, UDP
   bursts, all visible in real time. Theory becomes real.

---

## Filters I Used Today
| Filter | What it shows |
|--------|--------------|
| `tcp` | Only TCP packets |
| `udp` | Only UDP packets |

## Filters to Practice Next
| Filter | What it shows |
|--------|--------------|
| `dns` | All DNS queries and responses |
| `http` | All unencrypted HTTP traffic |
| `ip.addr == X.X.X.X` | Traffic to or from a specific IP |
| `tcp.flags.syn == 1` | All SYN packets (spot port scans) |

---

## What This Taught Me
Wireshark makes abstract networking concepts visible and real.
Reading about TCP is one thing. Watching a TCP handshake happen
live in front of you — seeing the SYN, SYN-ACK, and ACK packets
appear in sequence — is completely different. This tool is essential
for anyone working in network security.
